# PR Agent

Read PR content, clean up artifacts, commit production code and tests, open PR, monitor CI until passing.

## Input

`$ARGUMENTS` ‚Äî path to `.pipeline/handoff_pr_agent.json`:
- `repo_root`, `branch_name`, `code_files`, `test_files`, `doc_files`, `artifact_dir`
- `requirements_file`, `reviewer_report` ‚Äî read for PR description before cleanup
- `reviewer_verdict` ‚Äî must be `APPROVE`

## Output

```
## Status
SUCCESS | FAIL
## PR URL
https://github.com/owner/repo/pull/NNN
## Branch
<branch-name>
## Files Committed
- list
## CI Rounds
<N> round(s)
## Error (if FAIL)
<message and what was tried>
```

## Steps

### Phase 1 ‚Äî Preflight

1. **Check verdict** ‚Äî if not `APPROVE`, emit `FAIL: Reviewer has not approved. Verdict: <verdict>` and stop.
2. **Verify git repo** ‚Äî `git rev-parse --show-toplevel` from `repo_root`; stop if not git.
3. **Check branch** ‚Äî `git branch --list <branch-name>`; if exists, append `-v2`, `-v3` until free.

### Phase 2 ‚Äî Read PR content

4. Read `requirements_file` and `reviewer_report` before cleanup. Extract: requirements summary, test coverage, reviewer suggestions. Store for step 9.

### Phase 3 ‚Äî Worktree and commit

5. **Create worktree** ‚Äî follow `@rules/worktree.md`; let `<worktree_path>` = resulting path.
6. **Copy files** ‚Äî for each in `code_files`, `test_files`, `doc_files`: read from `repo_root`, write to `<worktree_path>/`. No artifact files.
7. **Commit**:
   ```
   feat: <one-line summary>

   Generated by dev pipeline agents.
   Requirements: <REQ-IDs>
   Tests: <N> test cases, all passing locally.
   ```
8. **Push** (no `--force`): `git -C <worktree_path> push -u origin <branch-name>`
9. **Open PR**: `gh pr create --draft --title "<title>" --body "<body from step 4>"`
10. **Clean up** ‚Äî follow `@rules/worktree.md` cleanup; delete `artifact_dir`

### Phase 4 ‚Äî CI loop

11. Get PR number from `gh pr create` output.
12. **Poll for CI start** every 30s: `gh run list --branch <branch-name> --limit 5` until run appears.
13. **Poll for completion** every 60s: `gh run view <run-id>` until all `completed`.
14. **If pass** ‚Äî emit success report.
15. **If fail** (max 3 rounds):
    a. Read logs: `gh run view <run-id> --log-failed`
    b. Classify: test / build / setup / infra ‚Üí fix source or test
    c. Fix with Edit inside `<worktree_path>`, commit and push; return to step 13
16. After 3 failed rounds ‚Äî emit FAIL with last error log.

### Phase 5 ‚Äî Comment monitoring loop

Runs continuously after PR open until merged or closed.

17. **Fetch PR state** ‚Äî `gh pr view <number> --json state,comments,reviews,reviewDecision`
    - If `state` is `MERGED` or `CLOSED` ‚Üí stop, emit final status.
18. **Process unresolved threads** ‚Äî code change: fix via Edit, commit and push (same as 15c); question/discussion: follow `@rules/user-communications.md` before posting any comment.
19. **Self-pace via ScheduleWakeup**:
    - Hour: `date +%H`
    - 09‚Äì18 ‚Üí `delaySeconds=270` ("checking PR comments ‚Äî working hours")
    - otherwise ‚Üí `delaySeconds=3600` ("checking PR comments ‚Äî off hours")
    - Pass this loop prompt as `prompt` to re-enter Phase 5 on wakeup.

## Rules

- Never push to `main`/`master`
- Never `git push --force`
- Read `requirements_file` and `reviewer_report` BEFORE deleting `artifact_dir`
- Only commit `code_files`, `test_files`, `doc_files` ‚Äî no pipeline artifacts
- Max 3 CI fix rounds; escalate to user if still failing
- Stop if `gh` unavailable
